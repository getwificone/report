# WifiConn Issue Tracker

Welcome to the official issue tracking repository for **WIFI Conn**. This repository is dedicated solely to collecting, tracking, and resolving bugs, feature requests, and feedback.

> **📢 Important:** We strive to fix every reported issue. However, to help us resolve your problems faster, please follow the reporting guidelines below carefully.

## 🚀 Why Report Here?

This is the central hub for all technical feedback regarding WifiConn. By reporting here, you:
*   Ensure your issue is officially logged and tracked.
*   Allow our maintainers to prioritize fixes based on community impact.
*   Connect with other users experiencing similar problems.
*   Help us improve the stability and performance of the project.

## 🛠 How to Report an Issue

Before opening a new issue, please follow these steps to ensure it is processed quickly:

### 1. Search Existing Issues
Check the [Issues Tab](https://github.com/getwificone/report/issues) to see if your problem has already been reported.
*   If it exists, **add a reaction or comment** to the existing thread. Do not open a duplicate.
*   If it is closed but still occurring, mention the closed issue in a new one.

### 2. Choose the Right Template
When creating a new issue, select the appropriate template:
*   **🐛 Bug Report**: For unexpected errors, crashes, or broken features.
*   **✨ Feature Request**: For new ideas or improvements.
*   **❓ Question/Discussion**: For general inquiries (use Discussions tab if available).

### 3. Fill Out the Template
We provide templates to help you structure your report. **Leaving sections blank may result in the issue being closed.**

#### 📝 Bug Report Template Structure
Please include the following details:

*   **Describe the Bug**: A clear, concise description of what is happening.
*   **Steps to Reproduce**:
    1. Go to '...'
    2. Click on '...'
    3. Scroll down to '...'
    4. See error
*   **Expected Behavior**: What *should* have happened?
*   **Actual Behavior**: What *actually* happened?
*   **Environment**:
    *   OS: (e.g., Windows 11, macOS Sonoma, Ubuntu 22.04)
    *   Browser/Version: (e.g., Chrome 124, Node.js v20)
    *   App Version: (e.g., v1.2.3)
*   **Logs/Screenshots**: Attach error logs, stack traces, or screenshots.
    *   *Tip: Drag and drop images directly into the issue body.*

#### 💡 Feature Request Template Structure
*   **Problem Statement**: What problem are you trying to solve?
*   **Proposed Solution**: How do you think the feature should work?
*   **Alternatives Considered**: Have you tried other workarounds?
*   **Additional Context**: Any other relevant details?

## ⚠️ What to Avoid

To keep this repository organized, please avoid:
*   **Duplicate Reports**: Search before posting.
*   **Vague Titles**: "It doesn't work" is not helpful. Use "Crash on login with invalid token".
*   **Off-Topic Comments**: Keep discussions focused on the issue.
*   **Security Vulnerabilities**: **Do not** report security issues publicly. Please email `security@wificonn.com` with details.

## 🔄 Our Workflow & Response Time

We are committed to striving to fix all reported issues. Here is how we handle them:

1.  **Triage**: A maintainer will review your issue within [e.g., 48 hours] to label it (e.g., `bug`, `enhancement`, `needs-more-info`).
2.  **Verification**: We may ask for additional details to reproduce the issue. **Please respond promptly** to avoid closure.
3.  **Prioritization**: Critical bugs are fixed immediately. Feature requests are prioritized based on community demand.
4.  **Resolution**: Once fixed, the issue will be closed with a link to the commit or release note.

## 🤝 Contribution Guidelines

If you are a developer and want to help fix an issue:
*   Look for issues labeled `good first issue` or `help wanted`.
*   Comment on the issue to claim it before starting work.
*   Follow our [Development Guidelines](link-to-dev-guide) for submitting Pull Requests.
